# 🛒 品出しアシスタント Pro - Demo

> 小売店バックヤード業務の品出し作業を、スマートフォン1つで支援するWebアプリ
>
> **これは公開デモ版です。** 実運用版は別リポジトリで認証付き限定公開しています。

🌐 **Live Demo**: https://shinadashi-pro-demo.vercel.app *(デプロイ後に更新)*

---

## 📖 プロジェクト概要

スーパーマーケットでアルバイトをしていた経験から、品出し作業における以下の課題を実体験しました。

- 手書きメモの紛失・転記ミス
- 新人が陳列場所を覚えるのに時間がかかる
- 「未運搬／運搬済」の管理がしづらく二度手間が発生する
- 売り場と倉庫を何度も往復することによる作業効率の低下

これらを解決するため、**現場で実際に使えるWebアプリ** を独学で開発しました。

本リポジトリは、就活・卒論用の **公開デモ版** です。実店舗のリアルなデータを掲載すると著作権・プライバシー上の問題が生じるため、**架空店舗「フレッシュマートあおぞら」** のデータに置き換えています。

---

## ✨ 主な機能

| 機能 | 説明 |
|------|------|
| 📱 商品ビジュアル一覧 | カテゴリ別タブで商品を画像と共に一覧表示 |
| ✅ タスクキュー方式 | 品出ししたい商品をタップで「タスク」に追加し、台車積載順に整理 |
| 🔍 多軸絞り込み | 陳列場所・カテゴリ・商品名検索の3軸で迅速にアクセス |
| 🔄 運搬ステータス管理 | 「運搬済 / 未運搬」で進捗を可視化 |
| 💾 ローカル永続化 | localStorage で端末ごとの作業状態を保存 |
| 📲 PWA対応 | スマホのホーム画面に追加可能、ネイティブアプリ風に動作 |

## 🛠 使用技術

### フロントエンド
- **HTML5 / CSS3 / Vanilla JavaScript (ES6+)**
- 軽量さと学習目的のため、フレームワーク非依存の構成を選定
- localStorage API による状態管理

### データ
- **CSV** をデータソースとして利用
- 静的データのため、運用が容易で更新コストが低い設計

### インフラ・運用（実運用版）
- **Vercel**: 静的サイトホスティング + Serverless Functions
- **GitHub Actions / Vercel自動デプロイ**: mainブランチへのpushで自動本番反映
- **Cookie認証**: HttpOnly + Secure + SameSite=Lax で30日セッション
- **環境変数**: パスワード等の機密情報はVercel側で管理

### デモ版（本リポジトリ）
- 実運用版の認証層を除いたミニマル構成
- 商品画像はSVGアイコンで自前生成（外部リソース依存ゼロ）

## 🔐 セキュリティ・プライバシーへの配慮

開発過程で以下の課題に直面し、それぞれ対応しました。

### 1. 著作権リスクへの対応
初期実装では商品画像を外部ECサイトから直接読み込んでいましたが、これが**ホットリンク（直リンク）** に該当し、著作権・利用規約上のリスクがあることに気づきました。本デモ版では、すべての商品画像を **SVGで自前生成** し、外部依存を排除しています。

### 2. 店舗・勤務先情報の漏洩リスク
画像のドメインから勤務先が特定可能な状態は個人情報・勤務先情報の漏洩リスクとなるため、デモ版では **完全架空のブランド・店舗名** に置き換えています。

### 3. CDNキャッシュとセッション認証の競合
実運用版では、当初Edge Middlewareによるサーバーサイド認証を実装していましたが、**CDNが認証後のHTMLをIP単位でキャッシュ**し、同一ネットワーク内の別デバイスから認証スキップが起きるバグに遭遇。`Cache-Control: no-store` でキャッシュを無効化し、認証チェックを **クライアントサイドJS + 専用APIエンドポイント** に分離する構成に変更することで解決しました。

これらの試行錯誤を通じて、**Web配信におけるキャッシュ層とアプリケーション層の責務分離** の重要性を学びました。

## 💻 ローカル実行
